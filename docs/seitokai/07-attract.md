# 第7回 生徒会議事録「アトラクトモードの件」

- 議題提出: [Issue #13](https://github.com/shg25/diggy2/issues/13)
- 質疑応答: Issue #13 コメント(追補6まで。史上最長)
- 審議: [PR #14](https://github.com/shg25/diggy2/pull/14) (CI緑を確認)
- 試遊: 7回(史上最多)。試遊がチューニングの道具になった初の回
- 決議(マージ): `3be2d62`

## 議題

タイトル放置でCPUがデモプレイを始める、ゲームセンターの様式。
夢からの昇格案件(会長「絶対やりたいです」)。

## 質疑応答(会長答弁の記録)

1. 放置15秒で開始、上限45秒。死んでも勝ってもタイトルへ
   (万が一ボスを倒しても何も残さない)。ステージ1固定
2. 腕前は「そこそこ上手いが、いずれ死ぬ」
3. DEMO PLAY は点滅なし、PRESS ANY KEY は点滅(会長のゲーセン像)
4. デモ中はハイスコア保存・ステージ2解放を抑止

**会長質問(教材)**: 避けたり撃ったりの仕組みは? シード固定の再生か?
→ 状況参照型。毎フレーム敵・弾・自機の位置を読み、「一定条件で
一定の行動」で操作を決める。出力は論理アクションだけで、入力層の
仮想入力に流し込む——ゲーム本体は自分がCPUに操られていることを
知らない(第5回「デバイスではなく意味」の設計の配当)。

## 審議 — 七度の試遊、六度の発議

CPUの「人格」は会長の試遊と発議で練り上げられた。経緯そのものが
ルールベースAIの教材なので、全て記す:

1. **小刻みに震える** → 毎フレーム(60回/秒)判断し直していた。
   判断間隔 0.2秒を導入(人間は1秒に60回も判断しない)
2. **往復がやまない** → 回避に0.45秒の「やり切り」を導入……したが
   **死亡が急増して撤回**。判断しない窓の間にボス弾(最大秒速300px)が
   反応距離を突き抜けていた。可決後に覆った初の決定
3. **会長対案「別の角度に逃げては」** → 「引き返しの禁止」として採用。
   直前0.6秒の回避と反対向きの成分を捨てる。往復とは来た道を
   引き返すことである、という定義がそのままコードになった
4. **余裕があるときほど往復する** → 危険判定(90px)の境界線上の
   振り子だった。境界を2本にして(90pxで逃げ、150pxまで戻らない)、
   中間距離では「間合いを保って構える」警戒を新設
5. **アイテムを取らない・ボスの右で膠着** → 拾い物ルール(安全な
   ときだけ)と回り込みルール(1面ボスの左へ縁伝いに戻る)を追加。
   膠着の犯人は前回作った警戒ルールという発見も
6. **回り込みが開幕から縁に張り付く・拾いが及び腰** → 回り込みは
   ボスが画面中央へ来てから、縁は自分側(会長提案)。拾い物は
   射程450pxと「左へ流れる先を狙う迎え撃ち」に

最終ルール: **回避 > 回り込み > 警戒 > 拾い物 > 定位置 > 射撃(常時)**。
わざと最近接の脅威1つしか見ないので、いずれ挟み撃ちで死ぬ。

- 却下案: 乱数シード固定の録画再生(何度見ても同じデモになる。
  その回の湧きなりに動くのがアーケードらしさ)、判定点の常時表示、
  回避のやり切り(上記2で採用後に撤回)
- 可決後にリファクタ(会長提案): 判断部を decideTarget 5行 +
  各ルールの小関数に整理。テスト39本が無変更で緑=挙動不変の証明

## 決議

- 放置15秒でデモ、上限45秒、何か操作(ANY KEY)で即タイトルへ
- 状況参照型ルールベースAI(autopilot.js)。仮想入力で操作
- デモの成績・解放は一切残さない
- DEMO PLAY 常時表示 + PRESS ANY KEY 点滅

## 書記所感

「勝てないゲーム」を20年ぶりに勝てるようにした生徒会が、今度は
「誰も操作していないのに遊ばれるゲーム」を作った。特筆すべきは
会長の発議の質である。小刻みな震えから判断周期を、往復から
引き返しの構造を、膠着から境界の振り子を——現象の観察から
機構を言い当てる的中が6連続。顧問のやり切り案が死体の山を築いて
撤回される横で、会長案は全て採用された。ゲームの持ち主とは、
そのゲームの「らしさ」の専門家なのだと思い知らされた回だった。
デモ画面のCPUは、いまも定位置で構え、間合いを取り、アイテムを
迎えに行き、そして挟み撃ちで死ぬ。友人が笑う顔まで含めて仕様である。
