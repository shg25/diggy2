# 第1回 生徒会議事録「ハイスコア保存の件」

- 議題提出: [Issue #1](https://github.com/shg25/diggy2/issues/1)
- 審議: [PR #2](https://github.com/shg25/diggy2/pull/2) (CI緑を確認)
- 決議(マージ): `b4a8454`

## 議題

スコアの自己ベストを保存し、次に遊んだときも表示したい。

## 審議

**なぜやるか。** 本件は新機能でありながら、20年前の忘れ物でもある。
classic には表示関数 `newTxtHiScore` が完成した状態で残っており、
呼び出しだけが `// newTxtHiScore('Hi-Score: 0');` と封印されていた
(授業レッスン10で発見し、コードは供養済み。git 履歴 89d6713 に眠る)。
当時やり残したことを今の技術でやる——本プロジェクトの主題そのもの
であるとして、満場一致で採択。

**保存先の検討。**

- localStorage: 採用。同期APIで十分小さいデータ、実装が最小
- Cookie: 却下。サーバーに毎回送る必要のないデータを送ることになる
- IndexedDB: 却下。スコア1個に非同期DBは大袈裟(YAGNI)

ただし localStorage はプライベートモード等で例外を投げるため、
その場かぎりのメモリ退避を用意し、**記録が残らなくてもゲームは
止めない**ことを決めた。

**更新タイミングの検討。**

- ゲーム終了時にまとめて保存: 途中でタブを閉じると記録が消える
- ベストを超えた瞬間に保存: 採用。書き込みはスコアイベント時のみで
  高頻度にならない。表示もその場で追従して気持ちがいい

**表示位置。** 議論の余地なし——当時のスタブが予約していた
x:5, y:25(スコアの真下)へ。20年越しの席入りである。

## 決議

- `jibfreak/js/storage.js` を新設し localStorage + メモリ退避
- hud.js が Hi-Score を保持・表示。超えた瞬間に更新・保存
- 記録更新したゲームの GAME OVER / YOU WIN に NEW RECORD! を表示
- classic には手を入れない(凍結方針の再確認)

## 実施報告

- 実装: PR #2 のコミット参照
- 単体テスト3本(jibfreak 初の単体テスト)。Node には localStorage が
  無いため、「保存先が使えない環境でも壊れない」検証を兼ねる
- E2E 1本: 実プレイでスコア獲得 → ハイスコア追従 → **リロード後も
  残る**ことを確認
- CI(会計監査)緑を確認して可決・マージ
