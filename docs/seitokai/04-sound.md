# 第4回 生徒会議事録「音を鳴らす件」

- 議題提出: [Issue #7](https://github.com/shg25/diggy2/issues/7)
- 質疑応答: Issue #7 コメント
- 審議: [PR #8](https://github.com/shg25/diggy2/pull/8) (CI緑を確認)
- 決議(マージ): `4f5cba4`

## 議題

JIB-FREAK MOBILE に効果音を付ける。RIDGE部の系譜、史上初の音。
(classic は完全に無音だった。エンジンの音機能は Flash 前提で
当時から使われていなかった)

## 質疑応答(会長答弁の記録)

1. **音源方式**: オシレータ合成を採用。会議中に「オシレータとは?」の
   質問があり、顧問より説明——指定周波数の波形を生成する部品で、
   インベーダーやファミコンの音源チップと同じ原理。会長「いいですね」
2. **場面**: 候補どおり。インベーダーのような質素な音で。
   勝利・NEW RECORD・ステージ2解放はテキスト表示と同時に
   「引き締まった・コンパクトな」ジングルを
3. **BGM**: 別議題。「キャラの動きが BGM 代わり」というインベーダー式にも
   触れたが、本作はキャラ数が多く合わないとの見解
4. **ミュート**: 既定は音なし。スマホ展開を見据え、設定は保存しない

## 審議

- **1978年と同じ原理で作る**: 録音素材を一切使わず、矩形波(硬い
  ピコピコ)・三角波(丸い音)・ノイズ(爆発)を Web Audio の
  オシレータで合成する。「レトロ風の音を真似る」のではなく、
  当時のハードと同じ方法で音を作る——GIFデコーダと同じ精神
- **自動再生制限**: AudioContext はミュート解除(Mキー)という
  ユーザー操作の中で初めて生成する。既定ミュートの仕様が、
  結果的にブラウザの制限と最も相性のよい設計になった
- **射撃音の音量**: 連射(押しっぱなし)で鳴り続けるため、
  他の音の1/3程度の控えめな音量にした
- **ジングルの重なり**: 勝利+解放+NEW RECORD が同時に起きうるので、
  0.5秒・1.0秒とずらして順に鳴らす
- **ポーズ中の沈黙**: ポーズ中はゲームイベント自体が起きない
  (update を呼ばない)ので、音も構造的に鳴らない。ガードを
  書くのではなく、構造が保証する

## 決議

- 効果音8種(射撃・撃破・被弾・アイテム・ボス登場・勝利・
  NEW RECORD・ステージ2解放)をオシレータ合成で
- M キーでミュート切替、既定OFF・保存なし。ヒント行に
  M: 音(ON/OFF) を表示
- BGM は別議題として持ち越し

## 書記所感

「オシレータとは?」という質問に答えるうちに、この方式が
単なる実装選択ではないことがはっきりした。ドット絵は自前
デコーダで、音は自前発振で——このプロジェクトは一貫して
「当時の作り方そのもの」を現代のブラウザに運んでいる。
質素な3音のジングルが鳴った瞬間、無音だった20年に
音が付いた。初めての音は、ドミソだった。
