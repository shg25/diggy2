# 第9回 生徒会議事録「職員室の設置の件」

- 議題提出: [Issue #17](https://github.com/shg25/diggy2/issues/17)
- 質疑応答: Issue #17 コメント
- 審議: [PR #18](https://github.com/shg25/diggy2/pull/18) (CI緑。文書と定義のみのため試遊対象外)
- 決議(マージ): `d7d2c47`

## 議題

複数のAIモデル・エージェントの役割分担を制度化する(通称: 職員室)。
第1回総会の「外部顧問の試験導入」を発展させ、実装面の分担まで設計する。
狙いはエージェント開発手法の学びとコスト管理の両立。

## 審議

- **「Fableが舵取り、実装は安いモデル」は本当か**(会長質問)への
  顧問見解: 概ね本当だが、効くのは仕様・テスト・規約が固い現場で、
  本リポジトリはまさにそれ。また最大のコストレバーは「セッションの
  基本モデル」であり、1つの会の中の細かい分業はサブエージェントの
  文脈読み直しコストでかえって損をする。分業が得なのは並列で
  独立した作業のみ
- **GPT(Codex)に実装させるか**: レビュー中心を採択。読み取り専用・
  並行で動き、Claude枠を消費しない——ChatGPT Plus 枠の最高の使い所。
  実装をまたがせると規約(日本語コミット・放課後ルール・debug窓のみ等)の
  共有コストが毎回かかるため、実装は Claude 側に一本化
- **常勤顧問の人選**(会長裁定): 期間限定クレジットのあるうちは
  Fable 5 を使えるだけ使う。厳しくなったら Opus 4.8 へ——リリース時に
  Sonnet でうまくいかなかった経験から、2段飛ばしの降格はしない。
  Sonnet は委員(範囲を切った並列作業)で実績を積んでから昇格判断
- **セキュリティの一幕**: 既存の Codex CLI は macOS のセキュリティ
  機構に実行バイナリを削除され半壊していた(会長がマルウェア警告を
  目撃。危険物検知か誤検知かは未判定)。残骸を撤去し、会長の手で
  公式最新版(0.144.6)を再インストール——今回は警告なし。
  「新品で警告が再発したら招聘を保留」を規約に明記
- 却下・保留した案: 実装のCodex分担(上記)、総称「職員室」以外の
  候補は特に挙がらず

## 決議

- 名簿: 常勤顧問 Fable 5(→いずれ Opus 4.8、会長判断)/
  外部顧問 Codex CLI(レビュー中心・読み取り専用サンドボックス原則)/
  委員 haiku・sonnet(並列で独立した作業のみ、.claude/agents/ に定義)
- 行事別: 生徒会=常勤が全工程+大PRは外部顧問レビュー、
  総会=交代後も Fable を呼び戻す価値、大掃除=点検3系統並列
- 実際にやってみて、必要に応じて調整する(会長方針)

## 書記所感

生徒会に職員室ができた。会長が「Claude Code からでも GPT を
呼び出せるのか」と確かめた瞬間が、この回の要だったと思う。
顧問が別の学校の先生に電話をかけて意見を聞く——擬人化すれば
それだけのことが、契約プランの枠・モデルの得手不得手・
サンドボックスの安全設計という現実の上に成り立っている。
半壊したCLIの撤去から再招聘までを含めて、外部の力を借りるとは
どういうことかの練習を、導入の日に一通りやれたのは幸運だった。
次の大掃除が、職員室の初仕事になる。
