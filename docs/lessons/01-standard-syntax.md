# レッスン01: 標準に従う — ブラウザ独自構文の罠

対応コミット: `3adc166`

## 何が起きたか

約20年ぶりにゲームを開いたら、画面は真っ白。原因は `js/ridge.js` のこの1行だった。

```js
function keyDown(keyCode) eventKeyDown(keyCode);   // SyntaxError!
```

関数の本体を `{ }` で囲まず式を直接書くこの書き方は「式クロージャ
(expression closure)」という **Mozilla(Firefox)独自の拡張構文**。
ECMAScript標準には一度も入らず、2018年にFirefox自身からも削除された。
当時Firefoxで開発していたから動いていただけで、他のブラウザでは
最初から動いていなかったはず。

修正は本体を波括弧で囲むだけ:

```js
function keyDown(keyCode) { eventKeyDown(keyCode); }
```

## 教え

1. **「自分の環境で動く」と「正しい」は別物。**
   目の前のブラウザで動いても、それが標準なのか独自拡張なのかを
   意識しよう。独自機能は、他の環境で動かないだけでなく、
   本家からもいつか消える。この1行はゲーム全体を20年後に
   起動不能にした。

2. **エラーは入口から読む。**
   画面が真っ白のとき、最初に見るのは開発者ツールのConsole。
   `SyntaxError` は「そのファイル全体が読み込まれていない」ことを
   意味するので、被害は1行では済まない。

3. **現代なら lint が一瞬で見つけてくれる。**
   `node --check file.js` や ESLint は、この種の問題を実行前に検出する。
   当時はなかったが、今は機械に任せられる検査は機械に任せよう。

## 余談

今回の調査では全JSファイルに `node --check` をかけて、
独自構文がこの1箇所だけであることを確認した。
エンジン側(lib/diggy/)は無修正で現代のChromeで動いた。
標準に沿って書かれたコードは長生きする、という良い実例。
