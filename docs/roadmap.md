# ロードマップ — 生徒会への議案書

新機能候補の一覧。生徒会([seitokai/](seitokai/))はここから議題を選んで
審議・実装し、議事録を残す。実装済みの項目は下部の「達成済み」へ移す。

対象はすべて **JIB-FREAK MOBILE** (classic は凍結)。
「触る層」は jibfreak の実モジュール名で記す。

## 議題候補

### 7. ステージ3を作る

- **内容**: 新しい敵と新ボスの追加ステージ
- **なぜ**: 定義テーブルの配当回収。「敵の追加=表に1行」を遊びとして実証。
  新規ドット絵も楽しい
- **触る層**: defs.js(表に行を足す) / stage.js(進行) / boss.js / gfx(素材)
- **規模**: 中〜大(素材制作しだい)

### 8. シューティングらしい当たり判定

- **内容**: 自機の当たり判定を見た目より小さく(弾避けの爽快感)、
  敵への攻撃判定は見た目どおりかやや大きく(当てやすさ)。
  ストレスを減らした分、敵の攻撃を激しくできる
- **なぜ**: 現在は classic 由来の「画像サイズ=判定」。現代の
  シューティングの文法に合わせ、手触りを一段引き上げる
- **触る層**: entity.js(isTouching に判定サイズの概念を追加) /
  defs.js(判定サイズを表に) / enemies.js・boss.js
- **規模**: 小〜中(バランス調整が本体)

### 9. アトラクトモード(ゲームAI)

- **内容**: タイトルで放置するとCPUがデモプレイを始める、
  レトロゲーの様式。「一定条件で一定の行動」のルールベースAI
  (AIエージェントではない)が自機を操作する
- **なぜ**: 会長「絶対やりたいです」(2026-07-14、夢から昇格)。
  友人が笑う顔まで含めて仕様である
- **触る層**: 新 `autopilot.js`(入力層に偽の入力を流し込む設計が
  素直) / main.js(タイトルの放置タイマー)
- **規模**: 中

## 夢(いつか)

- **オンラインランキング**: バックエンド必須。静的サイトの範囲を超える
- **リプレイ保存**: 乱数のシード固定が必要。ゲームループの深い改修
- **ゲームパッド対応**: Gamepad API。engine/input.js のみで済む
  設計だが、会長が現在パッドを持っていないため夢へ(2026-07-11)。
  「2P同時プレイの後でもいいかも」——2Pとの相性も良い
- **2P同時プレイ**: 1台のキーボードを左右で分け合う方式。
  第3回生徒会で会長より「タイトルのカーソル(arrow.gif)は当時
  2P同時プレイの選択画面を想定して作った気がする」との証言。
  20年前の構想の続きとして、いつか
