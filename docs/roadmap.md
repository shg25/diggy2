# 新機能ロードマップ(案)

v1.0(万全な旧作)を土台にした新機能フェーズの候補メニュー。
優先度と規模は目安。着手時は 1機能 = 1 Issue = 1ブランチ = 1 PR で進め、
CIが緑になってからマージする(このワークフロー自体が最初のレッスン)。

レイヤー構造(レッスン14)が整ったので、各機能に「どの層を触るか」を
併記できる。触る層が少ない機能ほど安全に作れる。

## 第1弾候補(小〜中粒・旧作の魂に沿うもの)

### 1. ハイスコア保存 ⭐イチオシ
- **内容**: スコアの自己ベストを localStorage に保存し、画面右上に表示。
  ゲームオーバー時に更新されたら演出
- **なぜ**: 20年前に作りかけて諦めた機能(レッスン10で亡骸を発見・供養済み)。
  「当時やり残したことを今の技術でやる」というこのプロジェクトの主題そのもの
- **触る層**: 基盤に `storage.js` 新設 / text(表示) / step(更新タイミング)
- **規模**: 小。最初のPR練習に最適

### 2. 音を鳴らす
- **内容**: 射撃・爆発・アイテム取得・ボス登場に効果音。Web Audio API で
  新モジュール `sound.js` を作る(エンジンの DGE.Audio は Flash 前提なので使わない)
- **なぜ**: 当時は無音だった。ゲームの手触りが一段変わる
- **注意**: ブラウザの自動再生制限(初回キー入力後に AudioContext を再開する)。
  音素材はフリー音源か自作
- **触る層**: 基盤に `sound.js` 新設 / sprite・step から発火
- **規模**: 中

### 3. ポーズ機能
- **内容**: P キーで一時停止/再開
- **なぜ**: 現代のゲームの最低限の礼儀。flow.js の状態機械に
  STEP_PAUSE を足す教科書的な練習にもなる
- **触る層**: flow / step / set(キー入力)
- **規模**: 小

### 4. タイトル画面の充実
- **内容**: 操作説明・ステージ選択のUI化(今は隠しコマンドSキーのみ)・
  ハイスコア表示
- **なぜ**: 初見の友人が説明なしで遊べるように。公開の前提整備
- **触る層**: step / text
- **規模**: 小〜中

## 第2弾候補(中粒・遊びを広げるもの)

### 5. ステージ3を作る
- **内容**: 新しい敵と新ボスの追加ステージ
- **なぜ**: 定義テーブル(レッスン04)と出現率定数(レッスン09)の
  おかげで、敵の追加は「表に1行足す」作業になった。その配当を
  遊びとして回収する。gfx の新規ドット絵も楽しい
- **触る層**: sprite(定義追加) / stage(進行) / gfx(素材)
- **規模**: 中〜大(素材制作しだい)

### 6. モバイル対応(タッチ操作)
- **内容**: 仮想パッド or スワイプ移動 + タップ射撃。画面スケーリング
- **なぜ**: 友人に共有するとき、スマホで開ける方が圧倒的に遊ばれる
- **触る層**: set(入力の抽象化) / index.html(viewport) / 新 `touch.js`
- **規模**: 中

### 7. ゲームパッド対応
- **内容**: Gamepad API で十字キー+ボタン
- **なぜ**: シューティングはパッドが気持ちいい。API も枯れていて簡単
- **触る層**: set / 新 `gamepad.js`
- **規模**: 小〜中

## 公開(機能と並行して)

### 8. GitHub Pages 2本立て公開
- **内容**: リポジトリをパブリック化し、Pages で
  **旧作(v1.0系)** と **新機能版** の両方を配信。
  例: `/`(ランディング) + `/classic/` + `/v2/`、または v1.0 タグを
  classic として固定配信
- **前提**: 済(ライセンス確認済・favicon解消済・README整備済)
- **規模**: 小(構成の意思決定が本体)

## 夢(いつか)

- **オンラインランキング**: バックエンドが必要。静的サイトの範囲を
  超えるので、やるなら別プロジェクト級
- **リプレイ保存**: 入力記録方式。乱数のシード固定が必要で、
  ゲームループの深い改修になる
- **canvas 移植**: エンジン作者も「DHTMLよりcanvasだった」と
  READMEに書き残している。ただしそれは「別のゲーム」を作る話に
  近いので、このプロジェクトの外側

## 推奨ルート

```
8(公開) → 1(ハイスコア) → 3(ポーズ) → 4(タイトル) → 2(音) → 6(モバイル) → 5(ステージ3)
```

先に公開してしまえば、以後の機能追加のたびに友人が
「おっ、更新されてる」を体験できる。開発の励みは最高の燃料。
