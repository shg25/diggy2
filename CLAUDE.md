# diggy2 開発規約

20年前のシューティングゲーム RIDGE部 の保存と、新版 JIB-FREAK MOBILE の
開発を行うリポジトリ。経緯は docs/lessons/、新機能は docs/seitokai/ を参照。

## 2つのゲームの扱い

- **classic/ (RIDGE部)**: 凍結。ゲームの挙動は当時のまま保存する。
  「勝てない」(goWin未配線)も仕様として温存が決定済み。
  コードを触るのは重大な問題があるときだけ
- **jibfreak/ (JIB-FREAK MOBILE)**: 進化する新版。自作canvasエンジン。
  classic の値・アルゴリズムを尊重しつつ、新機能や是正はこちらに入れる
- **lib/diggy/**: 借り物のエンジン。改変しない
- 素材(gfx/)は両ゲームで共有しない(凍結保護のため各自がコピーを持つ)

## ドキュメントの書き方

- **docs/lessons/ (授業)**: リファクタと移植の記録。全26回で完結済み。
  新しいレッスンは増やさない
- **放課後ルール**: 既存機能の修正・改善を行ったときは、その機能を
  扱ったレッスンの末尾に「## 放課後」節を追記する(なければ最も関連の
  深いレッスンへ)。番号は増やさず、コミットは通常どおり作る
- **docs/seitokai/ (生徒会議事録)**: 新機能は1機能=1回の会合=1議事録。
  形式は 議題/審議(却下案も残す)/決議/実施報告(対応コミット付き)/書記所感。
  README.md の一覧表に追記する。**Issue を立てたら実装に入る前に必ず
  会長(ユーザー)のレビューを待つ**(質疑応答は Issue コメントで)。
  完了条件はクローズまでに全チェックを埋める
- コミットメッセージは日本語。「なぜ」を本文に書く

## 検証

```sh
npm run lint / typecheck / test   # 速い検査(pre-commitでも自動実行)
npm run test:e2e                  # 実ゲームをChromeで遊ぶ回帰テスト(約3分)
```

- 挙動を変えたら必ず E2E まで回す。npm 出力をパイプするときは
  pipefail を付ける(終了コードの握り潰し事故が過去にあった)
- テストとの接点は window.ridge.debug (classic) /
  window.jibfreak.debug (新版) のみ。内部実装を直接触らない
- 自機無敵は debug.state.muteki = true

## デプロイ

main に push すると GitHub Pages が自動更新される。
本番: https://shg25.github.io/diggy2/ (トップ) → classic/ と jibfreak/。
挙動を変えるデプロイ後は本番でもスモーク確認する。

## ゲーム固有の知識

- 速度・確率などのバランス数値は「30fpsのフレームあたり」が単位。
  jibfreak では物理を30Hz固定tickで回して単位を保存している
- Diggy の角度規約は (270 - angle) して sin/cos (entity.js が互換実装)
- bg.gif は960px幅。canvas では2枚並べて描く
