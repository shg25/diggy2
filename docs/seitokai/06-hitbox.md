# 第6回 生徒会議事録「シューティングらしい当たり判定の件」

- 議題提出: [Issue #11](https://github.com/shg25/diggy2/issues/11)
- 質疑応答: Issue #11 コメント
- 審議: [PR #12](https://github.com/shg25/diggy2/pull/12) (CI緑を確認)
- 試遊: 会長がローカル試遊台で確認し「試遊OK」(**新設した試遊制度の初適用**)
- 決議(マージ): `2825bed`

## 議題

当たり判定を「画像の矩形=判定」から「役割に応じた判定サイズ」へ。
自機は避けやすく、敵には当てやすく——現代シューティングの文法を
取り入れ、将来の弾幕強化の土台を作る。

## 質疑応答(会長答弁の記録)

1. **自機の被弾判定**: 中央 8×8 で決定。「また調整するかも」との
   ことなので、数値は defs.js の表で変えやすくしておく
2. **判定の見せ方**: 通常プレイでは非表示、デバッグでのみ可視化
3. **敵への攻撃判定**: 各辺 +4px 甘くする。会長より「現在の仕様に
   なってから、たまに擦り抜けているように見えることがある」との
   体感報告。ボスの体当たり判定を狭める件も可決
4. **敵の攻撃強化**: 本議題には含めず、ステージ3等でバランスごと審議

## 審議

- **道具は1つ**: entity.js の centerBox(中心を保って判定サイズを
  差し替えた矩形を返す)。自機の縮小・敵の拡大・ボスの縮小、
  すべてこの1関数で表せる。isTouching は無改造
- **数値は表に**: defs.js の HIT_DEFS に自機 8×8 / 敵 +4px /
  ボス −16px を集約。バランス調整は表の1行で済む
- **向きごとの適用**: 自機被弾(敵・ボス・ボス弾 vs 中央8×8)、
  敵被弾(ショット vs 画像+4px)、ボス体当たり(画像−16px)。
  **アイテム取得だけは意図して従来どおり**——取り逃しストレスを
  増やさないため(items.js に注記)
- **猫バス問題**: 暴れモードで width/height が入れ替わるが、
  centerBox は現在の矩形の中心基準なので縦向きにも自動で追従
- 却下案: 判定点の常時表示(レトロな画面に異物)、hitbox を
  エンティティのフィールドに持たせる案(向きごとに使う矩形が
  違うため、呼び出し側で選ぶ方が正直)

## 決議

- 自機の被弾判定は見た目32×32の中央 8×8
- 敵の被弾判定は画像より各辺 +4px、ボスの体当たりは各辺 −16px
- アイテム取得は従来どおり
- `debug.state.showHitboxes = true` で判定矩形を可視化(デバッグ専用)
- 敵の攻撃強化は別議題へ

## 書記所感

今回から「試遊」が始まった。CI が緑でも、それは「壊れていない」
の証明であって「面白くなった」の証明ではない。かすった、避けた、
という手触りは数値では書けず、会長が実際に遊んで初めて可決される
——当たり判定という、まさに手触りそのものの議題が初適用だったのは
出来すぎというものである。可視化の赤い 8×8 を「ちょうど良い」と
言えるのは、20年この機体を操ってきた人だけだ。
