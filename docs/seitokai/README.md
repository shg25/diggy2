# 生徒会議事録

JIB-FREAK MOBILE の新機能は、生徒会で審議して実装する。
その様子を書記が議事録として記録したのがこのディレクトリである。

- **授業**([lessons/](../lessons/)) では過去のコードから学んだ。
  各レッスン末尾の「放課後」は、あとから直した居残り作業の記録
- **生徒会**(ここ)では未来の機能を審議し、実装する

## 議事録の形式

1機能 = 1回の会合 = 1つの議事録。各議事録には以下を記す。

- **議題**: 何を作るか
- **審議**: なぜ作るか、どう作るかの検討(却下した案も残す)
- **決議**: 決めたこと
- **実施報告**: 実装の説明と対応コミット
- **書記所感**: 会合を記録していて思ったこと

## 運用規則(GitHubを使った現代の開発フロー)

| GitHub | 生徒会 |
| ------ | ------ |
| Issue を立てる | 議題の提出 |
| ブランチで作業 | 分科会での検討 |
| Pull Request | 審議(差分が審議資料、CI が会計監査) |
| マージ | 決議・可決 |
| docs/seitokai/ | 書記の議事録 |

1議題 = 1 Issue = 1ブランチ = 1 PR。CI が緑になってからマージし、
マージ後に議事録を書く。

Issue の完了条件は、検証が済んだものからチェック \[x\] を入れ、
クローズ時には全項目が埋まっているのが原則。埋まらない項目が
残るなら、それは別の議題として切り出す(未達のまま黙って閉じない)。

## 会員名簿

- **会長**: ゲームの持ち主。議題の採否と方針を決める
- **顧問**: 実装を補佐する教員(授業のときの先生と同一人物)
- **書記**: この議事録を書いている
