# レッスン09: 数字に名前を — ただし全部ではない

対応コミット: `d844cdc`

## 主役: 定数を作った本人が定数を使い忘れた

このコードには `STEP_BATTLE = 13` というちゃんとした定数がある。
状態遷移はどこも `stepFlg === STEP_BATTLE` と書いてある。えらい。

なのに1箇所だけ、こう書いてあった。

```js
if ((num % 30) === 0 && stepFlg === 13) {   // 13…?
```

生の `13`。おそらく書いた日の自分は「13=ボス戦」を暗記していて、
定数を経由する必要を感じなかった。だが読む側(20年後の自分を含む)は
`13` を見て step.js の定数表と照合する羽目になる。
さらに怖いのは、いつか `STEP_BATTLE` の値を変えたとき、
**ここだけ置き去りになる**こと。定数の価値は「名前が付く」ことと
「変更が1箇所で済む」ことの両方にある。片方だけ使うと両方失う。

## 今回名前を付けたもの

| 生の数字 | 名前 | 意味 |
|---------|------|------|
| `13` | `STEP_BATTLE` | ボス戦状態(既存の定数を使うだけ) |
| `30` ×3 | `FPS` | ゲームのフレームレート |
| `600` ×2 | `BAN_DURATION_MS` | やられ演出(ban.gif)の表示時間 |
| `600` ×1 | `BOMB_DURATION_MS` | ボムの有効時間 |
| `180` | `PWR_SPAWN_RATE` | アイテム出現率(毎フレーム1/180) |
| `20` | `TEKI1_SPAWN_RATE` | 雑魚湧き率(毎フレーム1/20) |
| `3`, `4` | `PWR_DEFS.length`, `TEKI1_DEFS.length` | 種類数(表から導出) |

見どころが2つ。

**同じ600に2つの名前を付けた。** やられ演出の長さとボムの
有効時間は、値がたまたま同じなだけで別の概念。1つの
`DURATION = 600` にまとめると「ボムだけ長くしたい」日に
演出まで一緒に変わってしまう。レッスン06の裏返しで、
**同じ値 ≠ 同じ知識**。名前は値ではなく意味に付ける。

**種類数は数字をやめて表から導出した。** `Math.random() * 4` の
4は「雑魚が4種類」の意味なので、`TEKI1_DEFS.length` と書けば
レッスン04の定義テーブルに5種類目を足しただけで抽選にも自動で
入る。2箇所目の修正が不要になる構造ごと直せる例。

## 名前を付けなかったもの

2面ボスの回転制御には `600, 400, 480, 178, 270` といった座標が
並ぶが、これらは「画面の右端」「ボスの幅222を引いた位置」など
幾何的な計算結果だ。`BOSS_TURN_X3_LIMIT = 400` のような名前を
付けても読みやすくならない。こういう数字に必要なのは名前ではなく
**座標の図解コメント**か、いっそ「画面幅からの導出式」への
書き換えで、それは別のレッスンの仕事。

**「マジックナンバーは全部悪」ではない。** `x / 2` の2や、
`i + 1` の1に名前は要らない。基準は「この数字の意味を、
コードの文脈だけで読み取れるか?」。

## 教え

1. **数字は書いた瞬間だけ自明。** 意味は書いた人の頭の中にしかなく、
   頭の中は保存されない(20年後の自分で実証済み)。

2. **名前は値ではなく意味に付ける。** 同じ600でも意味が違えば別の
   定数。逆に、同じ意味なら値が同じでなくても近くに置く。

3. **数えられるものは数えさせる。** 種類数・要素数はコレクションの
   `.length` から導出すれば、データを足すだけでコードが追従する。

## 余談

調査中にデッドコードを発見: `bossFlg` は const.js で宣言されて
以来、誰にも読まれず書かれずに20年間鎮座していた。コメントアウトの
残骸もあちこちにある(`// jiki.center().show();` など)。
いずれ「消す勇気」のレッスンでまとめて掃除する。
