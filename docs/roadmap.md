# ロードマップ — 生徒会への議案書

新機能候補の一覧。生徒会([seitokai/](seitokai/))はここから議題を選んで
審議・実装し、議事録を残す。実装済みの項目は下部の「達成済み」へ移す。

## 大方針(第1回生徒総会で決議、2026-07-19)

**系譜で保存する。** classic 凍結の前例を、もう一段繰り返す:

```
RIDGE部 (2005, DHTML)            classic/  … 凍結済み。当時のまま
JIB-FREAK MOBILE (2026, canvas)  jibfreak/ … 大掃除で「静的サイトとして完成」→ 凍結
JIB-FREAK ONLINE (次章)          online/   … オンライン化はすべてこちら
```

オンラインのバックエンドが何年後かに死んでも、完成した静的ゲームが
丸ごと残る——凍結は最強の保存である。素材は凍結保護のため
各バージョンが自分のコピーを持つ(classic/jibfreak 間と同じ原則)。

## 次の行事: 大掃除

- **目的**: JIB-FREAK MOBILE を静的サイトとして完成させ、凍結に備える。
  現状を隅々まで見直し、細かい調整を尽くす
- あわせて**外部顧問(別AIレビュー)の試験導入**を行う。
  会長「技術的に早めに試したい。バックエンドが入ると
  オーケストレーションの価値は上がる。その前に試しておけるとよい」

## JIB-FREAK ONLINE の階段(発足後、上から順に)

| 段 | 内容 | 規模 |
| --- | ---- | ---- |
| ① 決定性 | シード付き乱数への置換・壁時計の追放。同じ入力列→同じ結果 | 小〜中 |
| ② リプレイ保存 | シード+入力列の記録と再生。①の検証装置を兼ねる | 中 |
| ③ 2機のゲームプレイ | まずローカル2P。20年前の arrow.gif 構想の回収 | 中〜大 |
| ④ オンライン協力 | ロックステップ+シグナリング(無料枠)。初の外部サービス | 大 |
| ⑤ オンラインランキング | リプレイ提出式(サーバー再生で検証=チート耐性) | 中 |

- 各段はそれ自体で完結した向上なので、途中で止まっても損しない
- **ステージ3(⏸保留: 素材担当の友人の作業時期待ち)** も ONLINE 側で
  再開する。新しい敵と新ボス。「敵の追加=表に1行」の配当回収。
  第6回決議の敵攻撃強化もここに合流

## 進行の指針

```
大掃除(MOBILE完成・外部顧問の試験) → 凍結 → ONLINE 発足 → ①〜⑤
```

小さく確実に積み、友人が「おっ、更新されてる」を体験する頻度を最大化する。
