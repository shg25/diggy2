# ロードマップ — 生徒会への議案書

新機能候補の一覧。生徒会([seitokai/](seitokai/))はここから議題を選んで
審議・実装し、議事録を残す。実装済みの項目は下部の「達成済み」へ移す。

対象はすべて **JIB-FREAK MOBILE** (classic は凍結)。
「触る層」は jibfreak の実モジュール名で記す。

## 議題候補

### 4. 音を鳴らす ⭐次回議題

- **内容**: 操作説明の表示・ステージ選択のUI化(現状は S キーの隠し操作)・
  ハイスコア表示(議題1の後)
- **なぜ**: 初見の友人が説明なしで遊べるように
- **触る層**: main.js(タイトル描画まわり)
- **規模**: 小〜中

- **内容**: 射撃・爆発・アイテム取得・ボス登場・勝利に効果音。
  Web Audio API で `engine/sound.js` を新設
- **なぜ**: 当時から無音だった。手触りが一段変わる
- **注意**: ブラウザの自動再生制限(初回入力後に AudioContext を再開)。
  素材はフリー音源か自作
- **触る層**: engine/sound.js 新設 / player・enemies・boss・main から発火
- **規模**: 中

### 5. モバイル操作

- **内容**: 仮想パッド or スワイプ移動。タップ開始と画面スケーリングは
  実装済みなので、残るは移動と射撃の操作系
- **なぜ**: MOBILE を名乗っているのにまだ矢印キーでしか動けない
- **申し送り(第2回生徒会)**: 画面上のポーズボタンもここで検討する
  (キー前提の操作はスマホで成立しないため)
- **触る層**: engine/input.js(論理アクションに流し込むだけで済む設計済み) /
  index.html
- **規模**: 中

### 6. ゲームパッド対応

- **内容**: Gamepad API で十字キー+ボタン
- **触る層**: engine/input.js のみ(論理アクション設計の配当)
- **規模**: 小

### 7. ステージ3を作る

- **内容**: 新しい敵と新ボスの追加ステージ
- **なぜ**: 定義テーブルの配当回収。「敵の追加=表に1行」を遊びとして実証。
  新規ドット絵も楽しい
- **触る層**: defs.js(表に行を足す) / stage.js(進行) / boss.js / gfx(素材)
- **規模**: 中〜大(素材制作しだい)

## 夢(いつか)

- **オンラインランキング**: バックエンド必須。静的サイトの範囲を超える
- **リプレイ保存**: 乱数のシード固定が必要。ゲームループの深い改修
- **2P同時プレイ**: 1台のキーボードを左右で分け合う方式。
  第3回生徒会で会長より「タイトルのカーソル(arrow.gif)は当時
  2P同時プレイの選択画面を想定して作った気がする」との証言。
  20年前の構想の続きとして、いつか
