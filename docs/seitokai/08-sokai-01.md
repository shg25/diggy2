## 次第

残る議題候補がステージ3のみ(素材担当の友人の作業時期が見えず保留の
公算)となったため、夢(いつか)を含めた全体方針を審議する。

## 審議

**会長発議「このゲームをオンラインゲームにできるか」**が総会の本題に
なった。オンラインランキングのバックエンド選定は手段の話であって、
先に決めるべきはゲームの拡張方針ではないか。2P同時プレイも、当時の
「1台のキーボードを左右で」は実際にできるケースが少なく、MOBILE を
名乗る本作ならオンライン協力を考えるべきでは。リプレイ保存も
つながる話では——という三題噺である。

顧問の技術見解: 本作は意外なほどオンライン化に向いている。協力プレイの
王道「ロックステップ」(入力だけを交換し、全員が同じシミュレーションを
手元で回す)の前提3つのうち、30Hz固定tickと論理アクションの入力層は
すでにあり(アトラクトモードが「入力を流し込めばゲームは操作者を
知らない」を実証済み)、欠けているのは決定性(シード付き乱数と
壁時計の追放)だけ。しかも決定性が入ればリプレイは「シード+入力列の
記録」、オンライン協力は「リプレイの入力列を相手からの実況に
差し替えたもの」、ランキングは「リプレイを提出させてサーバーで再生
検証(チート耐性)」——**三つの夢は同じ土台の上にあった**。
会長の直感どおりである。

これを受けた**会長の第二発議が総会の白眉**: 大掃除で MOBILE を
「静的サイトとして完成」させ、classic と同じように**凍結**し、
オンライン化は名前を変えた新作でやればよいのでは。トップの入口も
3つに。——顧問が「バックエンドが死んでも1人プレイは残る縮退設計を」
と構えていた問題を、設計ではなく**系譜**で解決する案であり、即採択。

- 却下・整理された案: 階段①②を MOBILE 側に入れる案(乱数置換は
  挙動が変わり得るため、凍結対象には入れない)、名前の
  「JIB-FREAK MOBILE ONLINE」(その場で ONLINE に刈り込み)

## 決議

1. **オンライン化を長期方針として採択**。階段は
   ①決定性 → ②リプレイ → ③2機のゲームプレイ → ④オンライン協力 →
   ⑤ランキング(リプレイ提出式)
2. **系譜で保存する**: MOBILE は次回の大掃除で完成→凍結。
   オンライン化はすべて新作 **JIB-FREAK ONLINE**(online/)で行う。
   名前は本総会で正式決定。素材コピーの凍結原則を踏襲
3. ステージ3は⏸保留のまま ONLINE 側の棚へ
4. 外部顧問(別AIレビュー)は次回大掃除で試験導入(会長「早めに
   試したい。バックエンドが入る前に」)
5. 夢: ゲームパッド(低優先)・BGM(追認)。取り下げ: 移動/射撃分離
   (会長「気になったら指摘する」)

## 書記所感

「JIB-FREAK ONLINE」という名前が決まった瞬間を記録できたのは
書記冥利に尽きる。RIDGE部が classic として凍結されたとき、それは
終わりではなく保存だった。MOBILE もいずれ同じ棚に並び、その隣で
新しいゲームが動き出す。バックエンドの寿命という現代の問題に、
このリポジトリは「完成品を凍結して系譜を増やす」という20年前から
続くやり方で答えた。凍結は最強の保存である——ロードマップに
書いたこの一行は、たぶんこのプロジェクト全体の主題である。
