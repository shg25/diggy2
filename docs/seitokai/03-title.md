# 第3回 生徒会議事録「タイトル画面の充実の件」

- 議題提出: [Issue #5](https://github.com/shg25/diggy2/issues/5)
- 質疑応答: Issue #5 コメント(書記による記録方式の初運用)
- 審議: [PR #6](https://github.com/shg25/diggy2/pull/6) (CI緑を確認)
- 決議(マージ): `4c1033f`

## 議題

初見の友人が説明なしで遊び始められるタイトル画面にする。

## 質疑応答(会長答弁の記録)

本会から、会長は会議の場で口頭答弁し書記が記録する方式に改めた。

1. ステージ選択は採用。ただし**ステージ2はステージ1をクリアすると
   選べるようになるアンロック制**とする——「わりと苦労してほしい」
2. UIはカーソル選択方式。レトロゲーの雰囲気重視
3. 操作説明は毎回固定表示。シンプルに

## 審議

- **カーソル素材**: gfx/title/arrow.gif を起用。20年前にタイトル用に
  描かれながら一度も使われなかった 4x7px の矢印。会長の証言では、
  当時は**2P同時プレイ**(1台のキーボードを左右で分け合う方式)の
  選択画面を想定していた可能性がある。この構想はロードマップの
  「夢」に記録した——20年前の構想は、まだ終わっていない
- **解放の永続化**: ハイスコアと同じ localStorage 方式。
  毎回クリアし直しでは「苦労」が過ぎるため
- **タイトルの S キー**: カーソルUIの導入により廃止。
  隠しコマンドは正面玄関ができたら退役する
- **未解放時の表示**: STAGE 2 を暗い色で見せて「クリアで解放」と
  添える。存在を隠すのではなく、目標として見せる(苦労の動機付け)

## 決議

- STAGE 1 / STAGE 2 の ↑↓ カーソル選択(arrow.gif)
- ステージ2アンロック制 + 解放演出「STAGE 2 が解放された!」
- 操作説明の常設表示
- classic には手を入れない

## 実施報告

- 実装: PR #6 のコミット参照
- E2E: 「ロック中はカーソルが動かない」「ステージ1のボスを実際に
  倒して解放 → ステージ2を選んで開始 → リロード後も解放が残る」を
  通しで検証(16秒かけて本当にクリアするテスト)
- CI 緑を確認して可決・マージ
